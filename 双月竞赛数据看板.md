# 易达安双月竞赛数据看板 产品需求文档

## 一、文档信息

版本 v1.0 日期 2026-06-15 状态 开发版，供前后端开发适用项目 易达安事故感知记录仪双月竞赛及入厂检测活动

本文档为产品需求说明，描述一个供集团统一查看竞赛数据的网页应用，用于收集各门店每日回传的统计表，监控回传进度，对接后端绑定与检测数据计算三项渗透率，并按品牌排名。

## 六、数据来源与结构

### 6.1 门店上传文件

新车安装统计表，单个 sheet，列依次为以下五项。

- 序号
- 日期
- 车架号
- 是否安装易达安记录仪
- 备注

日期可能是 Excel 序列号，也可能是文本。

售后安装统计表，简单版，单个 sheet，列依次为以下五项。

- 序号
- 进店日期
- 车架号
- 是否加装易达安记录仪
- 备注

售后入厂检测统计表，详细版，包含保养、维修与事故三个 sheet，列依次为以下七项。

- 序号
- 进店日期
- 维修类型
- 车牌号
- 车架号
- 是否加装易达安记录仪
- 备注

解析详细版时需读取全部三个 sheet 并合并。系统需要兼容简单版与详细版两种售后表，并把它们归为同一类入厂数据。

### 6.2 后端数据

绑定数据，至少包含以下字段。

- 车架号
- 绑定日期
- 所属门店

如能提供绑定渠道或装机账号角色，则更利于销售端与售后端的归属判定。

入厂检测数据，至少包含以下字段。

- 车架号
- 检测日期
- 所属门店
- 是否有检测照片

后端数据由网页通过接口实时拉取，作为分子来源。具体接口与字段见第十一节依赖项。

### 6.3 字段映射约定

| 用途 | 来源 | 取值 |
| --- | --- | --- |
| 新车销量分母 | 新车表 | 车架号去重计数 |
| 全天入厂台数分母 | 入厂检测表 | 车架号去重计数 |
| 入厂未安装车次分母 | 入厂检测表加后端绑定历史 | 进厂时无有效绑定的车架号去重 |
| 新增绑定分子 | 后端绑定数据 | 当期新增绑定车架号去重 |
| 实检台数分子 | 后端检测数据 | 有照片且落在门店入厂集合内的车架号去重 |

## 七、功能需求

### 7.1 文件上传与解析

1. 支持一次拖入多个 xlsx 文件批量上传。
2. 从文件名解析门店名、日期与表类型。门店名按管理员维护的清单与别名做匹配，表类型按文件名是否含新车或售后或入厂检测来区分。
3. 打开文件读取数据行，提取车架号、业务日期与是否加装等字段。详细版售后表需合并三个 sheet。
4. 日期归一化。Excel 序列号转日历日期，文本日期统一解析 2026.06.11 与 2026.6.11 等写法。
5. 车架号清洗，包括去空格，统一大小写，全角转半角，并做 17 位长度与字符校验。校验不过的行进入异常列表，不计入统计。
6. 文件可能是截至当日的累计台账，也可能只含当日数据，两种都要支持。系统按业务日期入库，每行包含所属门店、业务日期与车架号，并标记表类型。是否累计由数据本身决定，不靠文件名判断。重新上传时，对该次上传中出现的每个业务日期，用新数据整体替换该门店该表类型在该业务日期下的已有行，上传中未出现的业务日期保持不变。累计文件、单日文件，以及更正后的重传，都按此规则合并，不会重复计数。
7. 文件名无法匹配到门店或表类型时，进入待处理区，由管理员手工指定门店与类型后入库。

### 7.2 回传监控与当日完成判定

1. 按所选日期展示回传矩阵，行是门店并按品牌分组，列是新车表与入厂检测表两类，单元格显示已回传或未回传，并附上传时间与数据行数。
2. 顶部显示当日回传进度，例如已回传 23 家，共 35 家。两类表都到齐才算该门店当日完成。
3. 全部门店两类表均回传后，状态切换为当日回传已完成。
4. 未回传门店高亮显示，并提供导出未回传名单，便于催报。
5. 默认展示当日，可切换历史任意日期查看当日完成情况。

### 7.3 指标计算

1. 实时拉取后端绑定与检测数据，与已上传文件按车架号关联，计算每家门店三项率。
2. 计算区间为竞赛期内累计，台数按车架号去重。
3. 提供手动刷新，并支持按设定间隔自动刷新。每次刷新记录计算时间戳。
4. 每项率同时显示当前值与目标值，并标注达标或未达标。

### 7.4 排名展示

1. 按奥迪、林肯与红旗三个品牌分别排名，品牌之间互不混排。
2. 每个品牌下，对综合渗透率、入厂检测覆盖率与售后端渗透率分别从高到低排名，给出名次。
3. 同时提供一个可按任一率排序的明细表，每行包含以下内容。

- 门店名
- 三项率当前值
- 是否达标
- 数据最后更新时间

4. 排名页可生成只读公示视图。

### 7.5 导出

1. 导出当前排名表为 Excel。
2. 导出任意门店任意区间的车架号级明细，含分子分母构成，便于人工复核。
3. 导出未回传名单。

### 7.6 异常与对账

1. 列出解析失败的文件与行，含失败原因。
2. 列出数据不一致项，例如后端有检测记录但门店入厂表无此车架号，或门店标记已加装但后端无绑定记录。
3. 异常项支持人工标记处理状态。

## 八、计算规则细则

数据合并。计算前先合并各门店的上传，规则见 7.1 节第 6 条，得到该门店每类表在竞赛期内保留的全部数据行，再在此基础上去重计数。以下分子分母均指合并去重后的结果。

新车销量。该门店新车表合并去重后的车架号计数。

全天入厂台数。该门店入厂检测表合并去重后的车架号计数，同一车架号多次入厂只计一次。

入厂检测未安装车次。在入厂台数集合中，进店日期之前后端没有该车有效绑定记录的车架号去重计数。

新增绑定，全店口径。后端绑定数据中，绑定日期落在竞赛期内、所属门店为该店的车架号去重计数。

入厂新增绑定，售后端口径。新增绑定中，车架号落在该店入厂检测表、且绑定日期落在该次入厂时间窗内的部分，去重计数。

实检台数。后端检测数据中有照片，并且车架号落在该店入厂台数集合内的车架号去重计数。后端有检测但门店未报入厂的，或门店报入厂但后端无检测的，归入对账异常，不计入分子。

归属判定。车架号落在新车表归销售端，落在入厂检测表归售后端，两表都有的按绑定日期更接近售车日还是进店日归。

边界说明。综合渗透率在售后端装机较多时可能超过 100%，属既定口径，展示时不做截断。其余两项率正常情况下不超过 100%，若超过则提示对账异常。

## 九、非功能需求

性能。单次批量上传支持 50 个文件以内，单文件 5000 行以内，解析在可接受时间内完成。排名页在 35 家门店规模下秒级返回。

权限。按角色控制可见门店范围与是否可配置、可导出。

安全。车架号与车牌号属敏感数据，传输加密，仅授权角色可见明细。文件上传校验类型与大小，防止异常文件。

兼容。支持主流桌面浏览器。上传支持 xlsx，可选支持 xls。

可维护。以下均为可配置项，不写死在代码里。

- 门店清单与别名
- 品牌归属
- 竞赛期
- 目标值

界面。状态提示以中性样式为主，不大面积使用红色。

## 十一、数据模型与依赖项

### 11.1 主要实体

门店，字段如下。

- 编号
- 名称
- 品牌
- 文件名匹配别名
- 是否参赛

上传记录，字段如下。

- 门店
- 业务日期
- 表类型
- 原始文件名
- 上传时间
- 行数
- 解析状态

车辆行，字段如下。

- 门店
- 表类型
- 车架号
- 车牌号
- 进店或销售日期
- 是否加装标记
- 来源上传记录

后端绑定，字段如下。

- 车架号
- 所属门店
- 绑定日期
- 渠道或账号角色

后端检测，字段如下。

- 车架号
- 所属门店
- 检测日期
- 是否有照片

指标快照，字段如下。

- 门店
- 三项率取值
- 计算时间戳

### 11.2 后端对接依赖

本产品计算分子依赖后端提供两类数据，需后端团队确认以下任一对接方式。

1. 提供查询接口，入参为门店与日期区间，返回绑定记录与检测记录列表。
2. 提供定时导出文件，由本系统定时拉取。

绑定记录所需字段如下。

- 车架号
- 绑定日期
- 所属门店

检测记录所需字段如下。

- 车架号
- 检测日期
- 所属门店
- 是否有照片

若绑定记录能附带渠道或装机账号角色，可提升销售端与售后端归属的准确性。

## 十四、开发前需确认事项

以下事项已有结论或需在开发前与相关方敲定。

1. 竞赛期为 2026-06-01 至 2026-07-31，已确认。
2. 门店清单与品牌归属见附录 A。该名单依据上一轮竞赛排名表整理，本轮如有门店增减，以集团最终名单为准。路虎店归林肯品牌部，丰田店归红旗品牌部。
3. 后端两类数据的对接方式与字段，需后端团队确认可提供，详见 11.2 节，这是分子来源的前提。
4. 上传方式第一阶段为管理员集中上传，门店自助上传列入第二阶段，如需提前请在开发前说明。
5. 建议同时下发一份门店填报规范，明确必填门店与日期、车架号填写要求，以降低解析与归属错误。
