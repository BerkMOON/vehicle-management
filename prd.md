## 0. 业务口述需求（本期以本节为准）

> 来源：业务同事口述，供五哥团队催报、看排名用。  
> 与旧版 PRD v1.0 有差异处见 **§12 待确认**。

### 核心就四件事

1. **门店传表** — 管理员上传各店每日 Excel（新车 + 售后/入厂两类）
2. **催报** — **实时**看清今天**缺哪个门店**哪张表，**列表展示**，方便五哥他们去催
3. **记 VIN** — 每个文件上传解析完成后，立刻把**新车表**和**售后（入厂）表**里的车架号记下来
4. **算分排名** — 结合后台绑定/检测数据，**实时**算三个率，**按品牌**出 **3 张排名表**，网页展示 + **导出 Excel**

### 三个指标（业务口径）

统计区间均为 **6 月 1 日至当前**，台数按车架号去重。

| # | 指标 | 公式 | 分子来源 | 分母来源 |
| --- | --- | --- | --- | --- |
| 1 | **综合渗透率** | 6.1 至今日新增绑定 ÷ **销售端新车台次** | 后台绑定数据 | 新车表 Excel 车架号去重 |
| 2 | **入厂检测覆盖率** | 拍照完成入厂检测台次 ÷ **售后入厂台次** | 后台检测数据（有照片） | 售后/入厂表 Excel 车架号去重 |
| 3 | **售后渗透率** | 新增绑定与售后 VIN 重合台次 ÷ **售后入厂中没有易达安记录仪的台次** | 后台绑定 ∩ 售后 VIN 集合 | 售后表中「是否加装」为否（或等价语义）的 VIN |

**分子分母展开：**

| 名称 | 定义 |
| --- | --- |
| 销售端新车台次 | 该店新车表内，业务日期在 6.1~今日 的车架号去重计数 |
| 售后入厂台次 | 该店售后/入厂检测表内，业务日期在 6.1~今日 的车架号去重计数 |
| 6.1 至今日新增绑定 | 后台绑定记录：绑定日在 6.1~今日、门店匹配，VIN 去重 |
| 拍照完成入厂检测台次 | 后台检测：有照片，且 VIN 落在该店售后入厂集合内，去重 |
| 新增绑定与售后 VIN 重合 | 上述「新增绑定」中，VIN 同时出现在该店售后入厂集合内的台次 |
| 售后入厂中没有记录仪 | 售后入厂 VIN 中，Excel「是否加装易达安记录仪」为**否/未装**（或进店时后台无有效绑定，见 §12） |

### 排名怎么展示

- 出 **3 张排名表**，分别对应：综合渗透率 / 入厂检测覆盖率 / 售后渗透率
- 每张表内按 **奥迪、林肯、红旗** 三个品牌部分组，**组内**按渗透率从高到低排列（品牌间不混排）
- 页面实时更新；每张表支持 **导出 Excel**
- **输出格式以参考文件为准**：`易达安双月竞赛渗透率排名（11.1-11.30）.xlsx`（见 §0.1）

### 0.1 排名表输出格式（参考 Excel）

参考文件为上一轮竞赛结果样式，**本期 3 张表结构相同，仅列名不同**。

**表头：**

| 列 A               | 列 B     | 列 C     | 列 D     |
| ------------------ | -------- | -------- | -------- |
| 标题（合并 A1:D1） |          |          |          |
| （空）             | 分母列名 | 分子列名 | 比率列名 |

**三张表的列名：**

| 表             | B 列（分母）     | C 列（分子）         | D 列（比率）   |
| -------------- | ---------------- | -------------------- | -------------- |
| 综合渗透率     | 新车销量         | 记录仪加装量         | 渗透率         |
| 入厂检测覆盖率 | 售后入厂台次     | 拍照完成入厂检测台次 | 入厂检测覆盖率 |
| 售后渗透率     | 售后入厂未装台次 | 新增绑定重合台次     | 售后渗透率     |

**行结构（从上到下）：**

1. 奥迪品牌各门店（组内按渗透率 **降序**）
2. **奥迪小计**（B/C 列求和，D 列 = C 合计 ÷ B 合计）
3. 林肯品牌各门店 → **林肯小计**
4. 红旗品牌各门店 → **红旗小计**
5. **集团合计**（加粗 / 底色高亮）

**数值格式：**

- B、C 为整数台次
- D 为**小数比率**（如 `1.333333` 表示 133%），**不是带 % 的字符串**
- 渗透率可大于 1（综合渗透率常见，如加装量 116、销量 87 → 1.333333）

**标题示例：** `易达安双月竞赛渗透率排名（6.1-7.31）`（竞赛期起止月.日）

### 催报怎么展示

- 默认看 **今天**
- 用 **列表** 明确展示：哪些门店缺新车表、哪些缺售后/入厂表（未传高亮）
- 可导出未传名单

### 实时性

- **每上传并解析完一个文件** → 更新该店 VIN 池 → **立即重算**三项率与排名
- 同时可手动刷新；可选定时拉后台绑定/检测数据

## 3. 三条主链路

### 链路 A：收表 + 记 VIN

**两类表：**

| 表类型 | 文件名关键词 | 内容 |
| --- | --- | --- |
| 新车表 | 「新车」 | 1 sheet：序号、日期、车架号、是否安装易达安记录仪、备注 |
| 售后/入厂表 | 「售后」或「入厂检测」 | 简单版 1 sheet；详细版保养/维修/事故 3 sheet 合并 |

**解析要求：**

- 日期：Excel 序列号、`2026.6.11` 等统一转 `YYYY-MM-DD`
- 车架号：清洗 + 17 位校验；失败行进异常列表
- 文件名识别门店、表类型；失败 → 待处理区人工指定

**入库规则：**

- 按每行**业务日期**入库；重传时对该文件出现的每个业务日期 **整批替换** 该店该表该日期的旧数据
- 竞赛期统计时对保留行 VIN **去重**

### 链路 B：催报（本期重点）

```
看今天 → 列表展示缺传门店 → 导出名单 → 五哥去催
```

| 项       | 说明                                                   |
| -------- | ------------------------------------------------------ |
| 缺传列表 | 一行一家店：新车表 ✅/❌、售后表 ✅/❌；缺任一即未完成 |
| 完成标准 | 当天 **两类表都到齐**                                  |
| 汇总     | 顶部：已完成 X 家 / 共 Y 家                            |
| 可选矩阵 | 可与列表并存，按品牌分组查看                           |
| 导出     | 未回传门店 Excel                                       |

### 链路 C：算率 + 3 张排名表

```
新车 VIN 池 + 售后 VIN 池 + 后台绑定/检测 → 三个率 → 3 张排名表（各按品牌分组）
```

详见 **§0 三个指标**。

## 4. 数据从哪来

```
门店 Excel ──→ 新车 VIN / 售后 VIN（分母、催报、部分判定）
                    +
已有后台只读接口 ──→ 绑定、检测（分子，一期不新建存储）
                    ↓
              三个率 + 排名
```

### 4.1 一期：全部在前端浏览器完成（不上传表落库后端）

| 数据 | 一期方案 | 说明 |
| --- | --- | --- |
| 上传的 Excel 行 | **浏览器 localStorage** | 刷新页面数据仍在；换电脑/清缓存会丢 |
| 谁传了 / 缺传 | 由上传记录 + VIN 行推导 | 不依赖后端 |
| 分母（销量、入厂台次等） | 前端解析 Excel 计算 | ✅ 一期完整可用 |
| 分子（加装量、实检台次） | 前端**只读调用已有** admin 接口 | 不新建竞赛专用 API；见 §9.2 |

### 4.2 分子数据来源（只读，非「后端帮存上传表」）

| 分子 | 建议复用已有接口 | 字段 |
| --- | --- | --- |
| 记录仪加装量 | `GET /api/admin/device/business/list` 等 | vin、绑定/创建时间、store |
| 拍照完成入厂检测 | `GET /api/admin/device/entry-inspection-log/list` | vin、store、record_date |

一期**不需要**后端新增：上传记录表、车辆行表、竞赛配置表、指标快照表。

### 4.3 一期若暂不接只读接口

| 能力                                 | 状态                              |
| ------------------------------------ | --------------------------------- |
| 上传解析、记 VIN                     | ✅ 可用                           |
| 今日谁传了 / 缺传催报                | ✅ 可用                           |
| 排名表分母列（新车销量、售后入厂等） | ✅ 可用                           |
| 排名表分子列（加装量、实检等）       | ⚠️ 为 0，渗透率无意义             |
| 导出 Excel 版式                      | ✅ 可用（数值待接接口后才有意义） |

**结论：催报可一期完整交付；完整排名需接已有只读接口，或二期再接。**

## 5. 分期功能清单

### 5.1 一期（当前范围）— 前端为主，后端不存上传表

> **原则**：上传表、回传记录、VIN 池、指标计算逻辑都在**前端**；后端仅**可选**提供已有业务的**只读**数据算分子，**不**为竞赛新建落库接口。

| 模块 | 能力 | 实现位置 | 一期状态 |
| --- | --- | --- | --- |
| 文件上传 | 批量拖入 xlsx，解析新车/售后（含三 sheet 合并） | 前端 | ✅ 必做 |
| 记 VIN | 按门店+表类型+业务日期 replace 合并 | 前端 localStorage | ✅ 必做 |
| 催报 | 今日缺哪家、缺哪张表，列表展示 + 导出 | 前端 | ✅ 必做（五哥核心诉求） |
| 待处理区 | 文件名匹配失败时人工指定门店/类型 | 前端 | ✅ 建议 |
| 解析异常 | 非法 VIN、日期行列表 | 前端 | ✅ 建议 |
| 三项率计算 | 竞赛期内累计，按 §0 公式 | 前端 | ✅ 必做 |
| 3 张排名表 | 参考 Excel 版式，品牌内降序+小计 | 前端 | ✅ 必做 |
| 导出排名 / 缺传名单 | xlsx | 前端 | ✅ 必做 |
| 分子数据 | 绑定、入厂检测 | **只读**调已有 admin API | ⭕ 一期建议接；不接则分子为 0 |
| 配置 | 门店、别名、竞赛期、目标值 | 前端 localStorage | ✅ 必做 |

**一期不做的（后端）：**

- ❌ 竞赛上传文件 / 车辆行持久化 API
- ❌ 竞赛专用绑定/检测聚合 API（可二期统一封装）
- ❌ 服务端指标计算、快照、权限门店范围（二期）
- ❌ 门店自助上传
- ❌ 对账异常持久化、VIN 级明细导出、公示视图

**一期交付验收标准（最小）：**

1. 管理员上传两类表后，**当天缺传列表正确**
2. 分母列（新车销量、售后入厂台次等）与 Excel 一致
3. 三张排名表**版式**与参考 Excel 一致，可导出
4. （若接只读接口）分子与渗透率与手工抽样一致

### 5.2 二期 — 后端持久化与体验增强

| 模块                     | 说明                                  |
| ------------------------ | ------------------------------------- |
| 上传/VIN 落库            | 换浏览器不丢、多人共用、可审计        |
| 竞赛配置后台             | 门店名单、别名、竞赛期后台维护        |
| 封装竞赛查询 API         | 统一绑定/检测聚合，前端不再直连多接口 |
| 对账异常                 | Excel vs 后台不一致列表               |
| 公示只读页、VIN 明细导出 |                                       |
| 门店自助上传             |                                       |

### 5.3 三期 — 运营与权限

| 模块                     | 说明 |
| ------------------------ | ---- |
| 按角色可见门店、导出权限 |      |
| 定时快照、历史排名对比   |      |
| 服务端导出大报表         |      |

## 9. 一期能否用现有能力完成？

**可以。** 竞赛看板（`/dashboard/competition`）已按一期思路实现：

| 能力 | 现状 |
| --- | --- |

| 档位 | 内容 | 适用 |
| --- | --- | --- |

localStorage 限制：同一浏览器、不清缓存；生产长期使用建议二期落库。

## 11. 与旧版 PRD v1.0 的差异对照

| 项 | 业务口述（§0，本期优先） | 旧 PRD v1.0 |
| --- | --- | --- |
| **综合渗透率分母** | 仅 **销售端新车台次** | 新车台次 **+ 入厂未安装车次** |
| **售后渗透率分母** | 售后入厂中 **Excel 未装记录仪** | 进店日前 **后台无绑定** 的入厂 VIN |
| **售后渗透率分子** | 新增绑定 ∩ 售后 VIN | 新增绑定 ∩ 售后 VIN，且绑定日 ≥ 进店日 |
| **排名展示** | 明确 **3 张排名表** | 品牌分组 + 明细表，未强调 3 表 |
| **催报展示** | **列表** 缺哪家 | 矩阵为主 |
| **实时** | 每传完一个文件即重算 | 有刷新，未强调 per-file |
